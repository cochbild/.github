# Contributing

Thanks for your interest in contributing! Here's how to get started.

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your change (`git checkout -b feature/your-feature`)
4. **Make your changes** and test them
5. **Commit** with a clear message describing what you changed and why
6. **Push** to your fork and open a **Pull Request**

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Write clear commit messages (imperative mood: "Add feature" not "Added feature")
- Follow existing code style and conventions in the project
- Add or update tests when applicable
- Update documentation if your change affects usage

## Reporting Issues

- Use the **Bug Report** or **Feature Request** issue templates
- Search existing issues before creating a new one
- Include steps to reproduce for bugs

## Code Review

This is a solo-maintained project. I review PRs as time allows — it may take a while, but I appreciate your patience. I may request changes or suggest a different approach before merging.

## Questions?

Open a discussion or issue — happy to help!
